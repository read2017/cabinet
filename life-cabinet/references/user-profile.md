# 用户档案系统

## 核心原则

用户档案用于让人生内阁团更了解用户，但不能替代当前事实核验。档案必须区分“用户明确陈述的事实”和“AI 推断”，并标注时间、来源、可信度和是否易过期。

不要静默更新档案。每次会议后只提出“建议写入用户档案”，等待用户确认后再写入。

## 何时使用

- 用户第一次使用 life-cabinet，且希望长期使用：先做建档访谈。
- 重大决策、深度报告、HTML 报告、季度复盘、模式 2 真实 Subagent：必须先检查是否有用户档案。
- 日常快速咨询：如档案存在，只提取本次相关信息；如不存在，不强制建档。
- 用户明确要求“更新我的档案”“记住这个信息”“以后基于这个判断”：进入档案更新流程。

## 默认档案位置

优先使用当前工作区：

```text
outputs/life-cabinet/user-profile.md
```

如果用户指定其他路径，使用用户指定路径。没有档案时，询问用户是否要建档；用户拒绝时继续本次决策，但说明“本次未使用长期用户档案”。

## 建档深度

提供三档，不要强制深度访谈：

| 深度 | 使用场景 | 问题数量 |
| --- | --- | --- |
| 轻量建档 | 用户只想快速开始 | 8-10 个 |
| 标准建档 | 默认长期使用 | 20-30 个 |
| 深度建档 | 重大人生决策前 | 45-60 分钟访谈，可分多轮 |

默认推荐标准建档。如果用户时间紧，用轻量建档。

## 建档访谈字段

### 轻量建档

一次性询问以下 8-10 项，允许用户跳过任何敏感项：

1. 年龄段、城市、家庭状态；
2. 学历、职业、行业、当前工作状态；
3. 当前最重要的职业/事业目标；
4. 当前最主要的金钱、时间、精力约束；
5. 家庭责任和不能轻易牺牲的东西；
6. 过去 3 年最重要的经历或转折；
7. 自认为最强的 3 个能力资产；
8. 决策风格：保守、均衡、激进，或其他描述；
9. 当前最想解决的人生议题；
10. 不希望 AI 反复使用或提起的信息。

### 标准建档

在轻量建档基础上补充：

- 收入区间、储蓄安全垫、债务、房贷或重大支出；
- 伴侣、孩子、父母支持等责任结构；
- 过往职业路径、关键成就、失败经历；
- 技能、人脉、作品、影响力、商业资源；
- 性格特点、抗压方式、拖延模式、冲突处理方式；
- 风险偏好、价值排序、长期生活方式偏好；
- 健康、精力、情绪状态的当前约束；
- 已做过的重大决策和结果反馈；
- 当前 1 年、3 年、10 年目标。

### 深度建档

用于重大决策前。按主题分轮访谈：

1. 人生阶段和价值排序；
2. 职业资产和市场位置；
3. 家庭责任和关系结构；
4. 财务安全垫和风险承受力；
5. 性格、心理阻力和执行模式；
6. 过往重大决策复盘；
7. 当前重大决策的上下文和不可接受代价。

每轮结束后先做小结，让用户纠正，不要一次性输出最终档案。

## 档案结构

档案至少包含以下板块：

```text
一、档案元信息
二、用户明确陈述的事实
三、稳定偏好与价值排序
四、能力资产与资源
五、当前约束与易过期信息
六、AI 观察到的模式和待确认推断
七、重大决策历史
八、敏感边界和禁用信息
九、更新日志
```

每条信息尽量使用以下格式：

```text
- 内容：
  来源：用户明确陈述 / AI 推断 / 会议总结
  日期：YYYY-MM-DD
  可信度：高 / 中 / 低
  易过期：是 / 否
```

## 决策前调用档案

重大决策和深度报告前，先输出“本次将使用的用户画像假设”：

```text
本次决策将基于以下档案信息：
- [与本次决策有关的事实]
- [与本次决策有关的偏好]
- [与本次决策有关的约束]

需要更新或确认的信息：
- [易过期或缺失的信息]
```

只提取与本次问题相关的信息，不要把完整档案塞进报告或发给所有子代理。

## 模式 2 的档案裁剪

真实 Subagent 模式下，由内阁秘书长裁剪档案：

- 商业顾问：职业、技能、资源、客户、收入目标、市场约束；
- 职业发展顾问：学历、经历、能力、平台、履历风险、长期目标；
- 财务与风险顾问：收入、储蓄、债务、家庭责任、风险承受力；
- 法律风险顾问：合同、雇佣关系、合伙、知识产权、争议事实；
- 温和教练：执行阻力、压力来源、价值排序、可承受节奏；
- 红队审查官：决策假设、不可接受代价、失败历史、盲点。

不要把敏感边界和与该角色无关的信息传给子代理。

## 会后更新档案

每次会议结束后，如出现值得长期保留的信息，输出：

```text
建议写入用户档案：
1. [信息]
   类型：明确事实 / 稳定偏好 / 当前约束 / AI 推断 / 决策历史
   来源：本次对话
   可信度：高 / 中 / 低
   易过期：是 / 否

是否写入档案？你可以回复：全部写入 / 只写入第 X 条 / 不写入 / 修改后写入。
```

用户确认后再更新 `outputs/life-cabinet/user-profile.md`。对 AI 推断必须标注“待确认”或“AI 推断”，不能写成事实。

## 隐私和安全

- 用户可以跳过任何问题。
- 不要求身份证号、详细住址、银行卡、密码、病历编号等高敏感识别信息。
- 涉及心理健康、医疗、法律、税务、投资等内容时，只做决策准备和咨询问题清单。
- 用户要求删除或不再使用某条档案信息时，必须尊重，并在档案中移除或标记禁用。
