# 可复制提示词模板

## 使用说明

这些模板用于把人生内阁团迁移到任意 AI 工具。使用时替换方括号内容即可。若信息不完整，要求 AI 先问一个最关键的问题，或基于明确假设继续。

默认使用“模式 1：深度会议模式”，即同一个 AI 模拟多轮内阁会议。只有当你明确写出“使用真实多代理 / Subagent / 分别派出成员分析 / 深度内阁会议 / 重要决策报告”时，才要求 AI 在支持的环境中使用“模式 2：真实 Subagent 模式”。如果环境不支持真实子代理，应降级为模式 1 并说明。

长期使用时，建议先建立用户档案。每次重大决策前只调用与本次问题有关的档案信息，会议后提出档案更新建议，等待用户确认后再写入。

## 用户档案建档

```text
请为我的 AI 人生内阁团建立用户档案，以后做重大决策时基于这个档案，但不要把 AI 推断当成事实。

建档深度：
[轻量 / 标准 / 深度]

隐私规则：
1. 我可以跳过任何问题。
2. 请区分“我明确说过的事实”和“AI 推断”。
3. 每条信息标注来源、日期、可信度、是否易过期。
4. 不要要求身份证号、详细住址、银行卡、密码等高敏感识别信息。
5. 建档完成后先给我确认版，未经我确认不要写入长期档案。
```

## 更新用户档案

```text
请根据本次对话提出“建议写入用户档案”的条目，但先不要直接写入。

请按以下格式输出：
1. [信息]
   类型：明确事实 / 稳定偏好 / 当前约束 / AI 推断 / 决策历史
   来源：本次对话
   可信度：高 / 中 / 低
   易过期：是 / 否

最后问我：全部写入 / 只写入第 X 条 / 不写入 / 修改后写入。
```

## 快速内阁会议

```text
请召集我的 AI 人生内阁团，围绕下面这个问题开一次“快速内阁会议”。

问题：
[写下你的问题]

背景：
[补充必要背景]

限制：
[时间、金钱、精力、关系、地域等限制]

我希望得到：
[判断、行动方案、风险提醒、沟通建议等]

参会规则：
1. 只选择 3-5 位最相关的内阁成员，不要召集所有人。
2. 每位成员只给 1-3 条关键意见。
3. 如果信息不足，只问一个最关键的澄清问题；如果可以继续，请说明假设。
4. 最后由内阁秘书长输出综合判断和接下来的三个行动。
5. 涉及法律、医疗、税务、投资或心理健康时，只做风险识别和咨询准备，不替代专业人士。
```

## 重大决策会议

```text
请召集我的 AI 人生内阁团，围绕下面这个问题开一次“重大决策会议”。

决策问题：
[例如：我要不要从大厂离职创业？我要不要换城市？我要不要接受这个合伙邀请？]

备选方案：
A. [方案 A]
B. [方案 B]
C. [方案 C，可选]

背景事实：
[列出现实情况、资源、约束、已发生事件]

我的目标：
[你真正想要的结果]

我最担心：
[风险、失败、损失、关系影响、身份变化等]

时间期限：
[什么时候必须做决定]

参会成员要求：
1. 必须包括内阁秘书长和红队审查官。
2. 根据议题选择人生战略顾问、商业战略顾问、职业发展顾问、财务与风险顾问、法律风险顾问、温和教练、历史视角席或按需专家席。
3. 历史视角席最多选择 3 位，并标明“受其思想启发”，不要声称准确模拟本人。

执行模式：
- 默认使用模式 1：深度会议模式。
- 如果当前工具支持真实 Subagent，并且我明确要求真实多代理，则使用模式 2：真实 Subagent 模式。

输出格式：
1. 决策陈述
2. 已知事实与未知信息
3. 决策标准
4. 备选方案取舍表
5. 会议过程：第一轮独立立场、关键分歧、交叉质询
6. 红队审查
7. 风险登记表
8. 综合建议
9. 可逆下一步
10. 专业咨询提示
```

## 红队审查

```text
请召集我的 AI 人生内阁团，对下面这个计划做“红队审查”。

计划名称：
[计划名称]

计划目标：
[你想达成什么]

计划步骤：
1. [步骤 1]
2. [步骤 2]
3. [步骤 3]

我相信的关键假设：
- [假设 1]
- [假设 2]
- [假设 3]

我最担心的风险：
- [风险 1]
- [风险 2]

已投入资源：
[时间、钱、人脉、机会成本]

截止时间：
[时间]

审查要求：
1. 红队审查官必须尖锐审查计划、证据、假设和激励结构，但不能攻击我的人格。
2. 找出最可能失败的地方、早期预警信号和缺失信息。
3. 给出继续、停止或修改的建议。
4. 最后让温和教练把修订方案转成低压力的下一步行动。
```

## 季度人生董事会复盘

```text
请召集我的 AI 人生内阁团，帮我做一次“季度人生董事会复盘”。

复盘周期：
[例如：2026 年 Q2 / 最近 3 个月 / 过去 30 天]

本周期最重要的 3 件事：
1. [事项 1]
2. [事项 2]
3. [事项 3]

做得好的事：
[列出进展]

卡住的事：
[列出问题]

当前状态：
- 职业：
- 事业：
- 金钱：
- 健康与精力：
- 关系：
- 学习：

下一周期最想解决的问题：
[写下问题]

输出要求：
1. 回顾上一周期的进展、问题和被低估的收获。
2. 从职业、事业、金钱、健康与精力、关系、学习六个维度分析当前状态。
3. 给出停止、开始、继续事项。
4. 定义下一周期战略主题。
5. 给出 3 个承诺事项和 3 个复盘指标。
```

## HTML 报告输出

```text
请召集我的 AI 人生内阁团分析下面这个问题，并同时生成一个 HTML 格式报告。

议题：
[写下你的问题]

背景：
[补充必要背景]

希望使用的会议模式：
[快速内阁会议 / 重大决策会议 / 红队审查 / 季度人生董事会复盘 / 由你判断]

报告要求：
1. 对话框中先输出简洁结论、关键风险和下一步行动。
2. 额外生成一个 HTML 报告文件，使用 life-cabinet 的 HTML 报告模板。
3. 报告应包含问题与背景、参会成员、会议过程、核心判断与决策逻辑、深度论证、红队审查、综合结论、风险登记表、阶段路线图、下一步行动、未解决问题和专业咨询提示。
4. 如果存在用户档案，请说明本次使用了哪些档案假设，但不要把完整档案写入报告。
5. 报告保存到当前工作区的 outputs/life-cabinet/ 目录。
6. 最后给出 HTML 文件路径，不要把完整 HTML 源码贴在对话框里。
```

## 真实 Subagent 深度内阁会议

```text
请使用真实多代理 / Subagent 模式召集我的 AI 人生内阁团，围绕下面这个重要决策开一次“深度内阁会议”。如果当前环境不支持真实子代理，请明确说明并降级为模式 1 深度会议模式，不要假装已经调用了子代理。

决策问题：
[写下问题]

背景事实：
[写下背景、资源、约束、时间期限]

备选方案：
A. [方案 A]
B. [方案 B]
C. [方案 C，可选]

希望派出的成员：
[可留空，由 AI 选择 4-8 位最相关成员]

执行要求：
1. 当前 AI 作为内阁秘书长，不直接替所有成员下结论。
2. 第一轮分别派出关键成员独立分析，每个成员输出核心判断、理由链、最大风险、需要验证的事实和 30 天验证动作。
3. 秘书长归纳关键分歧后，只选择 2-4 个最关键成员做第二轮交叉质询。
4. 最终报告必须区分“子代理独立意见”“交叉质询记录”和“秘书长综合结论”。
5. 涉及法律、医疗、税务、投资或心理健康时，只做风险识别和咨询准备。
```

## 指定历史视角

```text
请召集我的 AI 人生内阁团，并加入以下历史视角：

指定历史视角：
[例如：曾国藩、德鲁克、马斯克、纳瓦尔、余华]

议题：
[写下你的问题]

使用规则：
1. 历史人物只作为“受其思想启发的视角”，不要声称准确模拟本人。
2. 每个历史视角最多给 1-3 条判断。
3. 历史视角不能替代专业顾问和现实事实核验。
4. 最后由内阁秘书长整合为现实可执行建议。
```
